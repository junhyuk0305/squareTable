## 시연 흐름

### Cue 1 (0:00 ~ 0:15) · 개막 한 문장

**화면:** 양쪽 다 첫 화면 정지 상태. 좌측 = 박지원 환영 화면, 우측 = 김영자 인박스 (미응답 0).

**발표자 멘트:**
> "이제부터 1분 30초 동안 보여드릴 건 단 하나입니다. **사장님 머릿속에만 있던 매장 노하우가, 알바 손 안의 앱에서 답변으로 튀어나오는 순간.** 그리고 답변이 없을 땐 어떻게 사장님께 다시 흘러가는지. 같이 보시죠."

**다음 트리거:** 좌측 화면 클릭으로 이동.

### Cue 2 (0:15 ~ 1:00) · 메인 시나리오 — 이물질 클레임

**화면:** 좌측 — 박지원이 입력창에 "머리카락 나왔대요" 타이핑 → 전송.

**발표자 멘트:**
> "알바 박지원 씨, 오픈 3주 차. 지금 매장에서 손님이 '머리카락 나왔다'고 컴플레인 중입니다. 평소라면 사장님께 전화하거나 멘붕이죠. 박지원 씨는 앱에 그대로 적습니다."

**화면 변화:**
1. 사용자 메시지 버블 ("머리카락 나왔대요") 우측 정렬로 뜸
2. 0.8초 후 AI 응답 4블록 카드 등장
   - **블록 1 — 지금 할 행동 (3단계):** ① 즉시 사과 + 음식 회수 ② 새 음식 무료 재제공 제안 ③ 사장님께 톡 (자동 메시지 템플릿 버튼)
   - **블록 2 — 말투 가이드:** "정말 죄송합니다, 바로 새로 만들어 드릴게요" — 변명/원인설명 금지
   - **블록 3 — 환불·교환 기준:** 음식값 전액 무료 + 다음 방문 5천원 쿠폰 (사장님 기준)
   - **블록 4 — 출처:** *"김영자 사장님 가이드 · 2026-06-12 음성 입력 · 신뢰도 92%"* (작은 글씨로 출처 칩)
3. 카드 하단에 👍/👎 버튼

**발표자 멘트 (응답 뜨는 동안):**
> "보세요. **AI가 일반론을 답하는 게 아닙니다.** 이 매장 사장님 김영자 씨가 지난주에 음성으로 직접 흘려넣은 노하우. 환불 기준도 이 매장 기준. 출처도 박힙니다."

**다음 트리거:** 좌측 뒤로가기 → 다른 시나리오 1개만 더 빠르게.

---

### Cue 3 (1:00 ~ 1:40) · 보조 시나리오 — 우유 떨어짐 (속도 강조)

**화면:** 좌측 — 박지원이 "우유 떨어졌어요" 입력.

**발표자 멘트:**
> "다른 질문도 보죠. **사소한 것도 다 받습니다.**"

**화면 변화:**
- 4블록 답변: ① 냉장고 2번 칸 예비 우유 위치 ② 없을 시 옆 매장 GS25 ③ 사장님께 발주 알림 자동 발송 버튼 ④ 출처: 김영자 사장님 · 2026-06-09
- 답변 도착까지 1초 미만

**발표자 멘트:**
> "1초. **이게 매장 회전율이 살 수 있는 속도예요.** 자, 그런데 — 진짜 핵심은 지금부터입니다."

**다음 트리거:** 좌측 뒤로가기, 입력창 클릭. 약간 무겁게 손을 멈춤 (긴장 만들기).

### Cue 4 (1:40 ~ 2:30) · 학습 순환 발화 — 정전 위기 ⭐

**화면:** 좌측 — 박지원이 천천히 입력: **"정전됐어요. 손님 5명 계신데 어떻게 해요?"**

**발표자 멘트 (강조, 톤 낮춤):**
> "오픈 3주 차 알바가 가장 두려워하는 순간. 사장님께 가르침 받은 적도 없고, 구글에 검색할 시간도 없습니다. 매뉴얼? 없죠. 이 질문, 우리 RAG에 매칭될까요?"

**화면 변화:**
1. AI가 0.5초 정도 타이핑 인디케이터 표시
2. **매칭 실패 화면** 등장 (일반 답변 X):
   - 큰 아이콘 ⚠
   - 메시지: *"이 질문은 아직 김영자 사장님께서 답해주신 적이 없어요."*
   - 버튼 1: **[사장님께 바로 물어볼게요]** (강조 — 큰 빨간 버튼)
   - 버튼 2: [지금은 일반 가이드라도 볼래요] (회색 작은 버튼, 발표자는 안 누름)
3. 발표자가 빨간 버튼 클릭
4. 좌측 상단에 토스트: *"김영자 사장님께 알림이 갔어요. 답변 오면 바로 알려드릴게요."*

**발표자 멘트:**
> "**핵심 결정타입니다.** 모르면 모른다고 합니다. 헛소리 안 합니다. 그리고 — 사장님 화면 보시죠."

**다음 트리거:** 시선·손짓을 우측으로 이동.

### Cue 5 (2:30 ~ 3:40) · 학습 순환 응답 — 사장님 인박스 ⭐

**화면:** 우측 — 김영자 인박스. 상단에 빨간 알림 배지 ⭕ "1" 펄스 애니메이션.

**발표자 멘트:**
> "사장님 김영자 씨. 오후 9시, 매장 마감하고 집 가는 길에 알림 하나 떴어요."

**화면 변화 (단계별):**

1. **알림 카드 등장**
   - 카테고리 칩: ⚠ Event (빨강)
   - 내용: *"박지원 알바님 질문: '정전됐어요. 손님 5명 계신데 어떻게 해요?'"*
   - 시각: 방금 전
   - 버튼: **[큰 마이크 버튼 🎙]** — 화면 절반 차지

2. **발표자가 마이크 버튼 누름**
   - 마이크 모드 진입, 파형 애니메이션
   - 발표자가 실제로 노트북 마이크에 또박또박 말함 (10~15초):
     > "정전나면 일단 손님들 진정시켜. 그리고 두꺼비집 확인하고, 그래도 안 되면 한전 국번없이 123 전화해. 그동안 손님들한테는 음료 서비스로 드리고, 5분 넘으면 환불해 드려."

3. **AI 객관식 위저드 자동 등장 (음성 → SQUARE 6칸 자동 채움 후, 빈칸만 객관식)**
   - 질문 1: *"손님 대기 시간이 몇 분 넘으면 환불?"* → ◯ 5분 ◯ 10분 ◯ 15분 (발표자는 5분 선택)
   - 질문 2: *"이 답변 공개 대상?"* → ◯ 박지원만 ◯ 모든 알바 (발표자는 모든 알바 선택)
   - **자유 텍스트 타이핑 0회** — 사장님 입력 부담 최소화 시연 포인트
   - 하단: [저장하고 박지원 알바님께 보내기] 큰 초록 버튼

**발표자 멘트 (위저드 진행 중):**
> "사장님은 타이핑 한 글자도 안 하셨어요. **말 한 번, 객관식 두 번.** 퇴근길 1분."

4. **저장 버튼 클릭**
   - 우측에 토스트: *"답변이 박지원 알바님께 전달됐어요. + 같은 질문이 또 오면 자동 답변됩니다."*
   - 인박스 알림 배지 0으로

**다음 트리거:** 시선을 다시 좌측으로.

### Cue 6 (3:40 ~ 4:20) · 학습 순환 닫힘 — 알바 화면 갱신 ⭐⭐

**화면:** 좌측 — 박지원 채팅 화면 그대로. 상단에 새 알림 배지 ⭕.

**발표자 멘트:**
> "박지원 알바님 화면. 정전 질문 던진 지 약 1분 후. **사장님 답변이 도착합니다.**"

**화면 변화:**
1. 채팅창에 시스템 메시지: *"김영자 사장님이 답변을 보내주셨어요"*
2. 4블록 답변 카드가 새로 렌더링 — Cue 2와 똑같은 포맷:
   - **블록 1 — 지금 할 행동:** ① 손님 진정 ② 두꺼비집 확인 ③ 한전 123 ④ 음료 서비스
   - **블록 2 — 말투 가이드:** 침착하게, "잠시만 기다려 주세요"
   - **블록 3 — 환불 기준:** 5분 넘으면 환불
   - **블록 4 — 출처:** *"김영자 사장님 가이드 · 방금 음성 입력 · 신뢰도 100%"*

**발표자 멘트 (가장 천천히, 가장 또렷이):**
> "이게 스퀘어테이블입니다. **시니어 노하우가 → 주니어 즉답이 되고 → 답변 없으면 다시 시니어로 → 학습 순환.** 한 번 들어온 답변은 다음 알바가 같은 질문 했을 때 자동으로 나옵니다. 매장 고유 RAG가 매일 자라요."

**다음 트리거:** 잠시 화면 그대로 두기 (3초 정적 — 청중에게 흡수 시간).

### Cue 7 (4:20 ~ 5:00) · 시니어 입력의 4 카테고리 보여주기 (추가 신뢰 빌딩)

**화면:** 우측 — 김영자 인박스 상단 탭 전환. 4 카테고리 칩 시각화.

**발표자 멘트:**
> "사장님이 흘려넣는 노하우는 단지 돌발 상황만이 아닙니다. 4 카테고리를 균등하게 받습니다."

**화면 변화:**
- 4 카테고리 칩 강조 (호버 시 카운트 보임):
  - 🔁 Routine — 청소·오픈·마감 절차 (5건)
  - ⚠ Event — 클레임·기기오류·돌발 (3건)
  - 📜 Context — 매장 룰·물품 위치 (2건)
  - 💡 Know-how — 제조 꿀팁·응대 (2건)
- 각 칩 클릭하면 해당 카테고리 PlaybookEntry 리스트 미리보기 (스크롤 1회만)

**발표자 멘트:**
> "마감 청소 어디까지 / 컵은 어디에 / 김치찌개 간 맞추는 법까지. **시니어가 30년 쌓은 거 다 받습니다.** 음성으로요."

**다음 트리거:** 좌측으로 돌아가서 마감 시나리오 1개만 빠르게.

## 핵심 청구점 매핑 (왜 이 순서인가)

| 청구점 | 어느 Cue에서 증명되나 |
|---|---|
| 시니어 노하우가 살아있다 | Cue 2 출처칩 ("김영자 사장님 가이드") |
| 주니어 즉답 (1초 미만) | Cue 3 우유 시나리오 |
| 답변 없으면 헛소리 안 함 | Cue 4 매칭 실패 화면 |
| 미답변 → 시니어로 흘러감 | Cue 4→5 좌→우 전환 |
| 시니어 입력 부담 최소화 | Cue 5 음성+객관식 (타이핑 0회) |
| 학습 순환이 닫힘 | Cue 6 좌측 자동 갱신 |
| 매장 자산화 | Cue 8 "1회 사용됨" 표시 |
| 카테고리 4종 균등 | Cue 7 4 칩 시각화 |

## 장애 폴백

### 폴백 A · RAG가 의도와 다르게 매칭 (Cue 2~3에서 일반답변 나옴)
**증상:** "머리카락 나왔대요" 입력했는데 매칭 실패 화면이 뜸 (시드 데이터 미로드).
**대응:**
1. 당황하지 말고 즉석에서 **"좋아요, 이게 정전 시나리오로 자연스럽게 넘어갑니다"** 로 멘트 전환
2. Cue 4를 앞당겨 진행 — 매칭 실패 자체가 핵심 청구점이므로 손해 없음
3. 데모 후 시드 데이터 재로드는 발표 끝나고

### 폴백 B · Cue 5에서 음성 인식 실패 (마이크 권한·소음·인식률)
**증상:** 마이크 눌렀는데 텍스트가 안 뽑힘 / 엉뚱한 단어로 뽑힘.
**대응:**
1. **"실제 매장에선 사장님 폰 마이크라 더 잘 됩니다, 노트북 마이크 한계가 있어요"** 한 줄 자조
2. 음성 위저드 우회 — UnknownQuery 상세 화면에 **[텍스트로 직접 입력]** 보조 버튼 클릭 (시연용으로만 노출, 실제 MVP는 음성 100%)
3. 미리 작성된 답변 텍스트 붙여넣기 → 객관식 위저드는 그대로 시연
4. 시간 손실: 약 20초

### 폴백 C · Expo Web 자체가 죽음 (브라우저 크래시·화면 동결)
**증상:** 좌 또는 우 창이 흰 화면 / 동결.
**대응:**
1. 즉시 **사전 녹화 30초 GIF** 재생 (사전에 슬라이드 11번 다음에 hidden slide로 삽입)
2. 멘트: **"화면 녹화로 대신 보여드릴게요. 실제 작동하는 영상입니다."**
3. GIF는 Cue 4~6 (학습 순환 핵심 구간)만 담은 30초 짧은 버전
4. 시간 손실: 영상 30초 + 멘트 정리 = 50초

### 폴백 D · 인터넷 끊김
**증상:** 와이파이 다운, LLM API 안 됨.
**대응:**
1. **걱정 없음 — 데모 앱은 로컬 SQLite 기반**, RAG 매칭도 로컬 임베딩, LLM은 답변 포맷팅에만 사용
2. LLM 호출 실패 시 자동으로 **로컬 템플릿 답변**으로 폴백 (4블록 구조는 유지, 자연어 매끄러움만 떨어짐)
3. 사전 리허설에서 비행기 모드로 1회 검증

### 폴백 E · 시간 초과 (Cue 6쯤 6분 넘김)
**증상:** Cue 1~6 진행했는데 이미 6분 경과.
**대응:**
1. Cue 7 (4 카테고리) **스킵** — 발표 슬라이드에서 한 줄로 커버
2. Cue 8 (마감 청소) **스킵 가능** — 학습 순환 닫힘은 Cue 6에서 이미 보였음
3. Cue 9 클로징은 절대 스킵 금지 — 가격·BM 한 줄이 마지막 회수 포인트

### 폴백 F · 시간 남음 (Cue 9에서 4분만 사용)
**증상:** 데모 너무 빨리 끝남.
**대응:**
1. Cue 7에서 카테고리별 PlaybookEntry 리스트 한 번 더 천천히 보여주기
2. 좌측 채팅 히스토리 보여주며 "이 매장에 지난주 들어온 질문들" 스토리텔링
3. 절대 즉흥으로 새 시나리오 만들지 말 것 — 사전 시드된 7건 안에서만 움직임

## 발표자 톤·태도 가이드

- **속도:** Cue 2~3은 빠르게, Cue 4~6은 천천히. 학습 순환 구간이 청구점 회수 구간.
- **시선:** 청중 7 : 화면 3. 클릭하는 순간만 화면 보기.
- **단어 금지 목록 (메모리 톤 가이드 반영):** "혁신", "AI 어시스턴트", "플라이휠", "마찰 제로", "Game changer" — 전부 금지. 대신 "학습 순환", "노하우 흘러들어옴", "입력 부담 최소화".
- **자랑 금지:** "RAG 정확도 92%" 같은 숫자는 출처칩으로만 보여주고 입으로 말하지 말 것. 청중이 발견하게 둘 것.
- **자기 검열:** "이게 진짜 신기하죠?" 류 자화자찬 금지. 화면이 신기하면 청중이 알아서 반응함.
